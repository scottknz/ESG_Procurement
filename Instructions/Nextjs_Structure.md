# Next.js App Structure

This file defines the canonical folder and file structure for the ESG procurement platform. The Gemini CLI must scaffold and maintain this structure exactly. Do not invent new directories without updating this file first.

---


## Mobile-First Component Strategy
Every component built with shadcn/ui and Tailwind MUST be mobile-responsive from day one. Suppliers frequently access platforms from mobile devices (e.g., consultants in the field checking bid status). Use responsive grid classes (`grid-cols-1 md:grid-cols-2`), hidden panels for data-heavy tables on small screens, and touch-friendly targets for all buttons and inputs.

## Build Order for Gemini

Build in this strict sequence. Do not skip ahead. Each step must pass its checks before the next begins.

0. **Initialise project:** Run `npx create-next-app@16.2` with App Router + TypeScript + Tailwind. Run `npx supabase init`. Install core dependencies: `@supabase/supabase-js`, `@supabase/ssr`, `stripe`, `@stripe/stripe-js`, `resend`, `@react-pdf/renderer`, `react-pdf-html`, `zod`, `shadcn/ui`, `novel`, `ai`, `@ai-sdk/google`.
1. `supabase/migrations/` ‚Äî Write all migration SQL from `Database_Schema.md`. Include all triggers for audit immutability, the partial unique index for org lead, and the `updated_at` triggers.
2. `src/types/database.ts` ‚Äî Regenerate after migrations with `npx supabase gen types typescript`
3. `src/lib/supabase/` ‚Äî Server + browser + admin clients
4. `src/lib/stripe/client.ts` ‚Äî Stripe server-side client, ¬£200 listing fee checkout, 5% invoice creation
5. `src/lib/validation/schemas.ts` ‚Äî Zod schemas for all key types and form inputs
6. `src/lib/audit/log.ts` ‚Äî Central audit helper (used everywhere)
7. `src/lib/tender/state-machine.ts` ‚Äî All tender status transitions with validation
8. `src/lib/tender/deadline.ts` ‚Äî Deadline lockout checker
9. `src/lib/messages/service.ts` ‚Äî Canonical tender message creation + email fan-out + communications_log writes
10. `src/lib/storage/unlock-check.ts` ‚Äî Checks `supplier_tender_unlocks` before serving documents to uninvited suppliers
11. `src/lib/scoring/calculate.ts` ‚Äî Weighted score formula and disqualification logic
12. `src/lib/background/scoring-queue.ts` ‚Äî Async AI evaluation job dispatcher (Supabase Edge Function trigger)
12.5. `src/middleware.ts` ‚Äî üî¥ ABSOLUTE: Auth guards, session refresh, and role-based redirects for all route groups. Must be built before any route group.
13. `(auth)` routes ‚Äî Login, register, callback, invite token onboarding
14. `(buyer)` routes ‚Äî Dashboard, wizard, evaluation, award, history, messages
15. `(supplier)` routes ‚Äî Dashboard, profile, bid form, unlock paywall, final review
16. `(public)` routes ‚Äî Storefront and public tender detail with reg-wall
17. `(admin)` routes ‚Äî Platform metrics, org management, audit log
18. `api/ai/` routes ‚Äî All Gemini AI integrations (server-side only)
19. `api/webhooks/stripe/route.ts` ‚Äî Handle ¬£200 listing fee + 5% invoice Stripe webhook events
20. `api/webhooks/deadline-lock/route.ts` ‚Äî Cron: moves tenders.status from 'open_for_proposals' to 'closed' when submission_deadline passes
21. `api/webhooks/scheduled-publish/route.ts` ‚Äî Cron: checks `publish_at` and moves tenders from 'draft' to 'open_for_proposals'. NOTE: You must also generate a `vercel.json` file to register the cron schedule for these two endpoints.
22. `api/webhooks/award-tender/route.ts` ‚Äî Post-award: generate audit PDF, ZIP archive, send emails
23. `src/lib/pdf/` ‚Äî Audit report + RFP export PDF generation
24. `src/lib/email/` ‚Äî Resend client + all email templates


25. **AI Copilot Integration (Critical - Both Sides)**: After all routes are built, implement the persistent buyer copilot:
   - `src/components/copilot/BuyerCopilot.tsx` ‚Äî left sidebar chat panel
   - `api/ai/copilot/route.ts` ‚Äî streaming function calling endpoint
   - Update `(buyer)/layout.tsx` to include copilot panel
   - Implement all function calling tools (getSubmissions, draftEmail, updateRFP, etc.)
   - Test context awareness: agent should know which tender and tab buyer is viewing
   - `src/components/copilot/SupplierCopilot.tsx` ‚Äî supplier left sidebar
   - `api/ai/supplier-copilot/route.ts` ‚Äî supplier assistance endpoint
   - Update `(supplier)/layout.tsx` to include copilot panel
   - Implement supplier function calling tools (draftAnswer, askClarification, checkReadiness, etc.)
